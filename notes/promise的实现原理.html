    # promise的实现原理

    首先声明以下

      // 判断变量否为function
  const isFunction = variable => typeof variable === 'function'
  // 定义Promise的三种状态常量
  const PENDING = 'PENDING'
  const FULFILLED = 'FULFILLED'
  const REJECTED = 'REJECTED'

    1、promise的状态具有以下特性

    A、promise存在 pending fulfilled rejected，状态只能从pending改为fulfilled或者rejected，且改变完不会发生变化 所以有以下代码

    class MyPromise {
constructor (handle) {
  if (!isFunction(handle)) {
    throw new Error('MyPromise must accept a function as a parameter')
  }
  // 添加状态
  this._status = PENDING
  // 添加状态
  this._value = undefined
  // 执行handle
  try {
    handle(this._resolve.bind(this), this._reject.bind(this)) 
  } catch (err) {
    this._reject(err)
  }
}
// 添加resovle时执行的函数
_resolve (val) {
  if (this._status !== PENDING) return
  this._status = FULFILLED
  this._value = val
}
// 添加reject时执行的函数
_reject (err) { 
  if (this._status !== PENDING) return
  this._status = REJECTED
  this._value = err
}
}

    # 2、promise的then方法

    promise的then接受两个参数，

    promise.then(onFulfilled, onRejected)

    onFulfilled、onRejected 都是可选的参数，但如果不是，其必须被忽略

    onFulfilled特性 A、当promise调用成功是被调用第一个参数是promise成功状态传入的值（就是resolve执行时传入的值） B、promise状态改之前不可被调用 C、其调用次数只有一次 onRejected 为失败，特性同上

    then 可以被一个promise对象多次调用

    A、当promise为成功时，所有的onFulfilled按照注册顺序一次执行回调 B、当promise为失败时，所有的onRejected按照注册顺序一次执行回调

    返回

    then返回一个新的promise对象 所以pormise支持链式调用
